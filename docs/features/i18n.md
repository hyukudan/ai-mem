## Configuration

### Environment Variable

```bash
# Set language
export AI_MEM_LANGUAGE=es

# Set fallback language (for missing translations)
export AI_MEM_FALLBACK_LANGUAGE=en
```

### Programmatic

```python
from ai_mem.i18n import set_language, get_language

# Set language
set_language("ja")

# Get current language
lang = get_language()  # "ja"
```

## Using Translations

### Basic Translation

```python
from ai_mem.i18n import t, _

# Translate a key
message = t("cli.add.success", id="abc-123")
# → "Memory stored! (ID: abc-123)"

# Shorthand
message = _("app.description")
# → "Universal Long-Term Memory for LLMs"
```

### With Variables

```python
# Variables are interpolated using {name} syntax
message = t("cli.search.results", count=5)
# → "Found 5 results"
```

### Specific Language

```python
from ai_mem.i18n import translate

# Translate to specific language
message = translate("app.name", lang="es")
# → "ai-mem"
```

## Translation Keys

### App
- `app.name` - Application name
- `app.description` - Short description
- `app.tagline` - Marketing tagline

### CLI Commands
- `cli.add.success` - Add success message
- `cli.add.error` - Add error message
- `cli.search.results` - Search results count
- `cli.search.no_results` - No results message
- `cli.context.generated` - Context generated
- `cli.session.started` - Session started
- `cli.endless.enabled` - Endless mode enabled

### Modes
- `modes.code.name` - Code mode name
- `modes.code.description` - Code mode description
- `modes.email.name` - Email mode name
- (etc.)

### Concepts
- `concepts.gotcha` - "Gotcha" translation
- `concepts.trade-off` - "Trade-off" translation
- `concepts.pattern` - "Pattern" translation
- (etc.)

### UI
- `ui.dashboard.title` - Dashboard title
- `ui.dashboard.search` - Search placeholder
- `ui.observation.type` - Type label
- (etc.)

### Errors
- `errors.not_found` - Resource not found
- `errors.invalid_input` - Invalid input
- `errors.unauthorized` - Unauthorized

### Messages
- `messages.loading` - Loading...
- `messages.success` - Success!
- `messages.error` - Error

## Adding Translations

1. Create a JSON file in `src/ai_mem/i18n/`:

```json
// src/ai_mem/i18n/pt.json
{
  "app": {
    "name": "ai-mem",
    "description": "Memória Persistente Universal para LLMs"
  },
  "cli": {
    "add": {
      "success": "Memória armazenada! (ID: {id})"
    }
  }
}
```

2. The language will be automatically available.

## Fallback Behavior

If a translation is not found:
1. Check current language file
2. Check base language (e.g., `pt` for `pt-br`)
3. Check fallback language (default: `en`)
4. Return the key itself

## Listing Languages

```python
from ai_mem.i18n import list_languages

languages = list_languages()
# [{"code": "en", "name": "English"}, {"code": "es", "name": "Español"}, ...]
```

## Best Practices

1. **Always use translation keys** instead of hardcoded strings
2. **Use variable interpolation** for dynamic values
3. **Provide English translations** as the base
4. **Test with RTL languages** (Arabic, Hebrew)
5. **Keep translations short** for CLI output
